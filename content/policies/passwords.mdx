# Password Policies

The password policy module lets you define and enforce password requirements for local admin accounts on your managed firewalls.

## What Password Policies Cover

Password policies help you enforce security standards across your fleet:

- **Complexity requirements** — minimum length, character class rules
- **Rotation schedules** — how often passwords should be changed
- **Account standards** — which accounts should exist, naming conventions

## Policy Hierarchy

Like all policy modules, password policies follow the three-level hierarchy:

1. **Global** — baseline password requirements for all devices
2. **Organization** — override for a specific client's devices
3. **Device** — override for a single device

## Compliance Checking

The platform evaluates each device's local admin accounts against the active password policy. Devices that don't meet the requirements are flagged in the [Compliance Checking](/policies/compliance) dashboard.
