# Security Services

The security services page provides fleet-wide visibility into the security features enabled on your managed firewalls.

## What You Can See

SonicWall firewalls offer a range of built-in security services. This page shows the status of each service across your entire fleet:

- **Intrusion Prevention (IPS)** — network-based threat detection and prevention
- **Gateway Antivirus** — malware scanning at the network perimeter
- **Anti-Spyware** — spyware and adware detection
- **Content Filtering** — web content category blocking
- **App Control** — application-level traffic control
- **Geo-IP Filtering** — country-based traffic blocking
- **Botnet Filtering** — known botnet communication blocking
- **Cloud Backup** — SonicWall cloud-based configuration backup
- **ATP (Advanced Threat Protection)** — sandboxing and advanced malware analysis

## Fleet-Wide View

Rather than checking security services device by device, this page aggregates the status across your fleet. You can quickly identify:

- Devices with critical security services disabled
- Inconsistent security configurations across similar devices
- Services that are licensed but not enabled
- Coverage gaps in your security posture

## Policy Integration

Security service requirements can be defined as policies. When you set a policy requiring specific services to be enabled, non-compliant devices are flagged in the [Compliance Checking](/policies/compliance) dashboard.
